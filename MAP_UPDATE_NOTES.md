# 町丁目詳細ページと地図操作

## 変更したファイル

- `app.js`：町丁目詳細ページ、MapLibreのクリック・ダブルクリック、キーボード操作、地図状態の保存と復元、初回操作案内
- `styles.css`：選択中の丁目、詳細ページ、操作案内、スマートフォン表示
- `data.js`：`mapItems` の各デモ情報に `areaId` を設定

`map-geo.js` は町丁目境界、人口、世帯数、面積の元データとして、そのまま利用しています。

## 記事や開発情報を町丁目へ紐づける方法

`data.js` の `mapItems` に `areaId` を入力します。

```js
{
  id: "example-item",
  title: "記事や開発情報のタイトル",
  areaId: "14215001201",
  targetArea: "中央一丁目"
}
```

`areaId` には、`map-geo.js` にある対象町丁目の `Feature.id` を文字列として入力します。画面表示用の町名ではなく、このIDを使って関連付けます。

中央一丁目のIDは `14215001201` です。URLは `#/areas/14215001201` になります。

## 地図から詳細ページを開く方法

- 町丁目を1回クリックまたはタップ：選択した丁目へ拡大し、右側に概要を表示
- 町丁目をダブルクリック：詳細ページを開く
- 右側の「この地域を詳しく見る」：詳細ページを開く
- キーボード：町丁目を選択し、Enterキーまたはボタンで詳細ページを開く

詳細ページへ進む前に、中心座標、ズーム率、選択中の町丁目ID、カテゴリーをブラウザ内へ保存します。「地図に戻る」またはブラウザの戻る操作で、直前の地図状態を復元します。

掲載中の地点、計画、記事はすべて画面確認用のデモデータです。

## 市全体図のランドマークを更新する方法

ランドマークは、変更内容ごとに次のファイルへ分けています。

- `city-landmarks-data.js`：Supabaseを使えないときに表示する8件の予備データ
- `assets/landmarks/`：繰り返し使えるランドマーク画像素材。SVG・PNG・WebPに対応
- 管理画面の `landmark-assets-manifest.js`：Gitにある画像を素材一覧へ表示する軽量台帳
- `city-landmarks.css`：イラスト、目印ジャンプ、ミニ全体図の見た目
- `app.js`：上記データを地図へ描画する共通処理
- 管理画面「ランドマーク素材」：素材を選び、地図上の地点へ配置・編集する画面

Supabaseの `city_landmarks` は「1行＝地図上の1配置」です。同じ `image_path` の行を複数作れるため、1つの画像素材を何地点にでも配置できます。通常の配置追加、名称・座標・並び順・公開状態の変更は、管理画面からこの台帳だけを更新します。配置を削除しても画像素材は残ります。画像本体はSupabaseへ保存しません。新しい絵柄を追加するときは `assets/landmarks/` に保存し、管理画面の `landmark-assets-manifest.js` に1件追加します。

画像ファイル名は半角英小文字・数字・ハイフンを使い、拡張子は `.svg`、`.png`、`.webp` のいずれかにします。全素材は、ログイン後の管理画面「ランドマーク素材」でGit管理の素材台帳から読み込みます。Supabaseから読むのは地図への配置一覧だけです。公開サイト側には素材一覧ページを置きません。

Supabaseのテーブルが未導入、空、または一時的に取得できない場合は、`city-landmarks-data.js` の予備データを表示します。
